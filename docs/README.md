# Math formulas
## Area
- Circle: S = πR²
- Rectangle: S = ab
- Square: S = a²

## Perimeter
- Circle: P = 2πR
- Rectangle: P = 2a + 2b
- Square: P = 4a

## Circle
```python
import math
'''
Модуль math предоствляет обширный функционал для проведения вычислений 
с вещественными числами(числами с плавающей точкой).
'''
def area(r):
    '''
    Принимает число, которое  является радиусом круга: r
    Возвращает площадь круга, вычисляя ее по формуле: π * r^2
    '''
    return math.pi * r * r
def perimeter(r):
    '''
    Принимает число, которое  является радиусом круга: r
    Возвращает периметр круга, вычисляя его по формуле: 2 * π * r
    '''
    return 2 * math.pi * r
```
### Example
```python
r = 2
print (f"Area = area(r)); Perimetr = {perimetr(r)}")


Area = 12.566370614359172; Perimeter = 12.566370614359172
```
## Rectangle
```python
def area(a, b):
    '''
    Принимает два числа, которые являются сторонами прямоугольника: a, b
    Возвращает площадь прямоугольника, вычисляя ее по формуле: a * b
    '''
    return a * b


def perimeter(a, b):
    '''
    Принимает два числа, которые являются сторонами прямоугольника: a, b
    Возвращает периметр прямоугольника, вычисляя его по формуле: 2 * (a + b)
    '''
    return 2 * (a + b)
```

## Square
```python
def area(a):
    '''
    Принимает число, которое является стороной квадрата: a
```

## Triangle
```python
def area(a, h):
    '''
    Принимает два числа, одно из них является стороной треугольника, другое его высотой: a, h
    Возвращает площадь треугольника, вычисляя ее по формуле: (a * h) / 2
    '''
    return a * h / 2
def perimeter(a, b, c):
    '''
    Принимает три числа, которые являются сторонами треугольника: a, b, c
    Возвращает периметр треугольника, вычисляя его по формуле: a + b + c
    '''
    return a + b + c
```

# Отчёт о тестировании функций расчета геометрических фигур

## Цели и задачи тестирования

### Цели тестирования:
1. Проверить корректность и точность функций для расчета площади и периметра (или длины окружности в случае круга) геометрических фигур: прямоугольника, круга, квадрата и треугольника.
2. Удостовериться в том, что функции корректно обрабатывают входные данные различного типа и диапазона значений.
3. Обеспечить стабильную работу функций в различных средах исполнения.

### Задачи тестирования:
1. Разработать и реализовать набор тестов для каждой функции.
2. Оценить соответствие реализации требованиям и спецификациям задачи.
3. Определить, как функции реагируют на крайние и нестандартные значения.
4. Протестировать интеграцию функций в итоговое приложение или библиотеку.

## Описание тестируемого продукта

Функции для расчета площади и периметра являются частью библиотеки или приложения, предназначенного для выполнения геометрических операций. Каждая функция принимает входные параметры, соответствующие фигуре (например, стороны для прямоугольника и треугольника, радиус для круга, сторона для квадрата) и возвращает численный результат.

## Область тестирования

### Функции для тестирования:
1. Функция расчета площади и периметра прямоугольника.
2. Функция расчета площади и длины окружности круга.
3. Функция расчета площади и периметра квадрата.
4. Функция расчета площади и периметра треугольника.

### Компоненты для тестирования:
- Алгоритмы расчета.
- Входные и выходные данные.
- Механизмы обработки исключений и ошибок.
- Производительность и оптимизация вычислений.

## Стратегия тестирования

Тестирование будет включать:
- Функциональное тестирование: Проверка каждой функции на соответствие с ожидаемым функциональным результатом.
- Тестирование граничных значений: Проверка поведения функций на краях допустимых значений входных данных.
- Тестирование ввода некорректных данных: Проверка реакции функций на некорректные или неожиданные входные данные.
- Тестирование производительности: Оценка времени выполнения функций и определение их производительности при больших нагрузках.
- Модульное тестирование: Изолированное тестирование каждой функции отдельно от остальных.
- Интеграционное тестирование: Проверка взаимодействия тестируемых функций с другими частями приложения.

## Критерии приемки

Продукт будет считаться готовым к выпуску, если:
1. Для всех функций достигнута точность расчетов в рамках определенной погрешности.
2. Выполнены все тесты, включая проверку граничных условий, и они показали удовлетворительные результаты.
3. Производительность функций находится на приемлемом уровне и соответствует ожиданиям.
4. Все найденные в процессе тестирования дефекты устранены или признаны несущественными для целей использования.

## Ожидаемые результаты

По завершению тестирования ожидается получение следующих результатов:
- Отчеты с детальным описанием каждого выполненного теста и его результатов.
- Список обнаруженных дефектов, их статусов и предложений по исправлению.
- Метрики качества кода и производительности.
- Окончательное заключение о соответствии продукта критериям приемки и готовности к использованию.
